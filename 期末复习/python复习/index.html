# python 中特有的数据类型，以及可哈希问题

# python 中的常用数据类型

  • 内置数据类型：Number（数字）、String（字符串）、List（列表）、Tuple（元组）、Set（集合）、Dictionary（字典）
  • 数字类型： int（整型）、float（浮点型）、complex（复数类型）

# 数据类型

  • 列表
  • 元组
    元组中的元素不能修改，可以有重复元素

  • 集合
    集合中的各元素无序、不允许有相同元素且元素必须是可哈希（hashable）的对象。

  • 字典
    键必须是唯一的，即不同元素的键不能相同；另外，键必须是可哈希数据，即键不能是列表、元组、集合、字典等类型；值可以是任意类型

# 可哈希（hashable）

  • 可哈希对象是指拥有__hash__(self) 内置函数的对象。列表、元组、集合和字典类型的数据不是可哈希对象，所以它们不能作为集合中的元素。
  • 简要的说可哈希的数据类型，即不可变的数据结构
    • 数字类型（int，float，bool）字符串 str、元组 tuple、自定义类的对象
    • 相同的值存储于相同的内存，不同的值存储于不同的内存，这和哈希表的性质一致。
    • 比如说 int，两个相同的值，在内存中的存储地址是一致的。
    • 不可哈希的类型（list，set，dictionary），更改值后，对象本身还是存储在同一个地址上。

# python 的参数顺序

  • 普通参数：可以用位置参数形式传递，也可以用关键字参数形式传递
  • 带默认值的参数：必须在普通参数后面
  • *args：接受 0 个或多个参数，args 是一个 tuple 当向函数中传递 tuple 时，用 * 修饰
  • 只能用关键字参数形式传递的参数：如果在 * args 后，仍有普通参数或默认值参数，只能用关键字参数形式传递
  • **kwargs：接受 0 个或多个关键词参数，位置不限，但必须在位置参数的后面，kwargs 本身是一个字典，向 kwargs 中传递字典中元素时，用两个 * 修饰

  • global 用于在函数内对函数外对象操作
  • nonlocal 关键字用来在函数或其他作用域中使用外层 (非全局) 变量。

# 类方法和静态方法的区别

# 类方法

  • 类方法的第一个参数是 cls（类），用 @classmethod 修饰
  • 类方法通常用于涉及到类的对象或属性，但不只属于某一个对象的操作

# 静态方法

  • 静态方法不需要 self 或 cls 参数，用 @staticmethod 修饰
  • 静态方法通常用于不涉及类内的属性或方法，但和类有关的操作

# 控制对于变量的访问

使用 @property 装饰器修饰变量

  • 使用 @property 来定义一个用于获取属性值的方法 (getter)
  • 使用 @属性.setter 装饰器来定义 setter 方法，在 setter 方法中可以设定对变量赋值的限制
  • 如果一个属性只有 @property 而没有 setter，它就是一个只读属性
  • 在 setter 和 getter 方法中访问属性，必须在属性名前加下划线，否则会不断递归访问导致报错

# super 方法

  • super 方法用于获取父类的代理对象，以执行已在子类中被重写的父类方法
  • super 本身其实是一个类，调用它实际上是产生了一个 super 对象
  • 代理对象：父类只被调用一次
  • super ([类名 [, 对象名或类名]]) 一般为 super (cls, self)
  • super 获取的是参数中类 cls 在实例 self 的方法解析顺序列表（mro) 中的下一个类的代理对象
  • 在调用父类的构造方法时，实际会把子类构造出的对象 self 传递给父类的构造方法，并一级一级向上传递，这样就保证了在多级继承的情况下，mro 表始终不变

# 文件指针的位置

  • 在打开文件时，如果打开方式中不包括 'a'，则文件指针指向文件首的位置；随着读 / 写操作，文件指针顺序向后移动，直至读写完毕。
  • 如果打开方式中包括 'a'，则文件指针指向文件尾的位置，此时向文件中写数据时就会在已有数据后写入新数据。

# 设定读写位置

  • file.tell ()：输出文件读取位置
  • file.seek ()：设置文件读取位置，以字节为单位

# 不同打开方式

  • 'r+' 或 'rt+'：以文本方式打开文件并可以对文件进行读 / 写操作。文件不存在会报错
  • 'w+' 或 'wt+'：以文本方式打开文件并可以对文件进行读 / 写操作。文件不存在会新建文件，文件已存在则会清空文件内容
  • 'a+' 或 'at+'：以文本、追加方式打开文件，可对文件进行读 / 写操作。文件不存在会创建新文件，文件已存在则文件指针会自动移动到文件尾
  • 'rb+'：与 'r+' 类似，只是以二进制方式打开文件
  • 'wb+'：与 'w+' 类似，只是以二进制方式打开文件
  • 'ab+'：与 'a+' 类似，只是以二进制方式打开文件

# 迭代器和生成器

# 迭代器

迭代器有两个基本的方法：iter () 和 next ()，字符串，列表或元组对象都可用于创建迭代器

  • 可以自定义类的内置函数 iter，来实现迭代器
  • 迭代器在迭代完成后会抛出 StopIteration 异常
  • 因此可以自定义类的内置函数 next，使其在符合条件时主动抛出 StopIteration 异常来结束迭代

# 生成器

  • 把列表生成式的 [] 改为 ()，可以创建一个生成器
  • 当函数中使用了 yield 时，称为生成器
  • 生成器是一个返回迭代器的函数，只能用于迭代操作
  • 在调用生成器时，每次遇到 yield 时函数会暂停并保存当前的运行状态，返回 yield 的值，并在下一次调用迭代器的 next 函数时从当前位置继续运行
  • 如果 yield 前是赋值操作（a = yield 5），那么如果调用 next 函数，会直接从下一条语句继续执行，而不会进行赋值操作，此时需要用 send 函数，传入函数一个值，才能赋值给 a

# csv 模块

  • csv 模块的 writer 方法可以生成一个 writer 对象，使用该对象可以将数据以逗号分隔的形式写入到 CSV 文件中。
  • 如果将 open 函数返回的文件对象作为实参传给 csvfile，则调用 open 函数打开文件时必须加上一个关键字参数 “newline=''”。
  • csv 模块的 reader 方法可以生成一个 reader 对象，使用该对象可以将以逗号分隔的数据从 CSV 文件读取出来。
  • open 函数返回的文件对象除了是可迭代对象，同时也是迭代器。如果将文件对象作为实参传给 csvfile，则调用 open 函数打开文件时应加上一个关键字参数 “newline=''”。
  • 返回的 reader 对象是一个可迭代对象，因此可以使用 for 循环直接遍历 CSV 文件中的每一行数据，每次遍历会返回一个由字符串组成的列表。
