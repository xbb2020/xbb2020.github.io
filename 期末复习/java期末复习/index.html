# 输入 / 输出流 9 章

# 流的分类

  • 按处理的数据单位：字符流，字节流

    字符流处理单元为 2 个字节的 Unicode 字符，分别操作字符、字符数组或字符串，而字节流处理单元为 1 个字节，操作字节和字节数组。所以字符流是由 jvm 将字节转为 2 个字节的 Unicode 字符为单位的字符而成的，所以它对多国语言支持性比较好。

  • 按数据流方向：输出流，输入流

  • 缓冲流：先将数据写入缓冲区，再由底层流把数据写入目的地

  • 数组流：使用数组（存储在计算机内存中）作为源和目标

  • 字符串流：使用字符串作为输入流的源，将内存作为输出流的目的地

  • 数据流：数据流以其他流作为底层流，可以读取与机器风格无关的 java 原始数据

  • 对象流：以对象作为源或目的地

# 掌握常用的类和接口的使用

  • File 类
  • FilenameFilter 接口

如果向 list () 方法传入 FilenameFilter 对象，则在调用时，参数会不断地调用对象的 accept () 方法，将目录中的文件名依次作为 name，dir 为调用 list 的当前对象，当方法返回值为 true 时，文件就会被存入返回值数组
accept 方法中通常会判断文件名是否符合某个条件 **（正则表达式）**

  • Scanner 类
  • FileReader 类 文件字符输入流
  • FileWriter 类
  • BufferedReader 类

  1. 以文件名作为参数构造文件流对象
  2. (可选) 将文件流作为底层流构造一个流对象
  3. 用流对象读取或写入文本文件

# 线程 8 章

# 常见创建线程的两种方式

  • Thread 类的子类
  • Runnable 接口
    java 只能单继承，所以继承 Thread 类后就不能再继承其他类，因此可以通过实现 Runnable 来继承其他类

# 线程的生命周期

  • 线程的四种状态
    • 新建：一个线程被新建出来还没有运行
    • 运行：线程调用 start 方法后，jvm 为线程分配 CPU 使用权时，线程开始运行
    • 中断 ：jvm 将 CPU 资源分配给其他资源，线程执行 sleep () 休眠，线程执行 wait () 时，线程执行操作引起阻塞时
      • wait(), notify(), notifyAll()
    • 死亡：线程的 run () 方法运行完成或强制结束，线程释放内存

# 面向对象 4,5 章

# 类、抽象类及接口的创建

  • 类
    • class name{}
  • 抽象类
    • abstract class A{}
    • 抽象类可以有抽象方法，也可以有非抽象方法
    • 抽象方法只允许声明，不允许实现，而且不允许用 final 和 abstract 同时修饰一个方法
    • 抽象类不能用 new 直接创建对象
    • 抽象类的非抽象子类必须必须重写全部抽象方法，这样可以让抽象类对象为子类上转型对象
  • 接口
    • interface B{}
    • 接口中只有对方法的声明
    • 接口通过其他类来实现，实现类必须重写接口中的所有方法
    • 接口中的方法默认为 public abstract 修饰

# 构造方法

  • name(){}
  • 一个类可以有多个构造方法，但参数必须不同

# 重载含义

  • 方法重载是指一个类中可以有多个方法具有相同的名字，但这些方法的参数必须不同，即参数的个数或类型不同
  • 返回值不参与比较

# 多态的实现

  • 上转型，子类对象的类型可以为父类，此时只能执行子类中重写父类的方法

# final、super、this 等关键字的含义和应用

  • final
    • final 修饰类，类不能被继承
    • final 修饰方法，方法不能被重写
    • final 修饰变量，如果是数值变量，那么数值在初始化后不能修改，如果是引用变量，则在初始化后不能指向其他变量

# Java 中常见的修饰符

  • 访问修饰符
    • public：本类、同包、子类和全局 (只要导入包就能访问)
    • protected：本类、同包和子类
    • 默认：本类、同包
    • private：本类
  • 其他修饰符
    • static：静态
      • static 修饰类时只能修饰内部类
      • static 不能修饰局部变量
    • final：同上
    • abstract：抽象
    • synchronized：为方法加锁，同时只能有一个线程调用该方法
