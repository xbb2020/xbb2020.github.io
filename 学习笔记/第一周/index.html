# 第一周学习笔记

# 学习内容

算法部分会常驻学习内容，但不会详细介绍

# 学习总结

# shiro 的作用

  • 验证用户身份
  • 用户访问权限控制
  • 相应认证，访问控制

# shiro 的特点

  • 易于使用
  • 灵活
  • 低耦合
  • 适用范围广

# 特性

  • 认证：Authentication
  • 授权：Authorization
  • 会话管理：Session Management
  • 加密：Cryptography

# Shiro 认证过程

  1. 创建 SecurityManager
  2. 提交认证
  3. SecurityManager 认证
  4. Authenticator 认证
  5. Realm 验证

# Maven 依赖

和 redis 整合时可以同时添加 redis 依赖

# 测试类

以上例子中，Realm 使用的是直接模拟的账号，实际使用时会使用数据库数据自定义 realm

当结果为 true 时表示验证成功，false 为失败
流程详解：

  1. 构建一个 SecurityManager 环境，shiro 默认 DefaultSecurityManager
  2. 设置 SecurityManager 的 Realm，用来验证账号
  3. 把设置好的 SecurityManager 传给 SecurityUtils
  4. 获取 SecurityUtils 的验证主体
  5. 创建一个包含用户名和密码的 token
  6. 使用 token 登录
  7. login 方法会自动调用 SecurityManager 来验证用户名密码是否包含在 Realm 中
  8. 如果验证成功，结果为 true，否则为 false

# 授权过程

这个例子中使用的同样是直接设置的 Realm

# 加密

当直接使用明文作为用户名和密码时，信息可能会被盗取，所以要将信息加密后再进行存储，加密的方法通常为 md5 + 盐 + 多次加密
此处细节内容会和整合 Redis 的部分一起补充

# 算法部分简单总结

本周算法的学习内容主要是两场训练中欠缺的部分

Set: 集合，可以快速查找元素是否已经存在，也可以便捷的初始化
Mutilset: 可重复集合，集合中的元素可以重复，可以使用 count () 来统计每一个元素的个数
